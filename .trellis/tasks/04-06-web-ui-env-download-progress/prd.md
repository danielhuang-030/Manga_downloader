# Web UI：`.env` 管理、啟動下載、即時進度（SSE）

## 目標

提供僅限本機的 Web UI（**FastAPI**），讓使用者可編輯與漫畫下載相關的 `.env`、啟動下載，並透過 **Server-Sent Events（SSE）** 觀看**即時進度**；同時保留既有 **CLI 進入點**（`main_env.py`），與 Web 共用同一段下載編排邏輯。

## 需求

- **設定**
  - **v1 以專案根目錄 `.env` 為唯一設定來源，不引入資料庫**（與 `python main_env.py`／`load_manga_config()` 一致）。
  - 讀取並顯示目前 `.env` 中與 `load_manga_config()` 相關的鍵值（與現行相同：`MANGA_COOKIES`／`BOOKWALKER_COOKIE`、`MANGA_RES`、`MANGA_SLEEP_TIME`、`MANGA_IDS`，選填 `MANGA_VIEWER_URL_TEMPLATE`、`MANGA_HEADLESS`）。
  - 將編輯結果寫回專案根目錄的 `.env`，並做驗證（盡可能沿用 `manga_env` 解析器與 `load_manga_config` 規則）；實作上採原子寫入與鍵級合併等策略以提升可靠性（見設計稿）。
  - **機密**：Cookie 不得出現在伺服器日誌；讀取 API 可選擇回傳遮罩後的 Cookie（選填），但儲存時須如實保留使用者意圖（細節見設計稿）。

- **下載**
  - 從 UI 觸發的下載行為須與「以環境變數為主的 CLI」一致（viewer ID + 非 `.env` 欄位沿用 `main.py` 預設）。
  - **v1 僅允許一個進行中的下載**（第二次啟動要拒絕或排隊——實作時擇一；預設：**拒絕**並回傳明確錯誤）。

- **進度**
  - 使用 **SSE**（非 WebSocket）將結構化進度串流至瀏覽器。
  - 事件至少涵蓋：執行開始、每書／每 viewer 邊界、頁面進度（已知時的目前／總頁數）、完成、失敗（錯誤文字須安全，不洩漏敏感資訊）。

- **相容性**
  - `python main_env.py` 須持續可用；共用邏輯集中於可被 CLI 與 API 呼叫的小型編排模組。

## 驗收準則

- [ ] FastAPI 應用程式預設僅綁定 **127.0.0.1**（連接埠可設定，預設值須有文件說明）。
- [ ] 使用者可透過 UI 載入、編輯、儲存 `.env`，且不破壞無關行（若可行則保留註解／順序；若不可行，文件須說明最小合併策略）。
- [ ] 使用者可從 UI 啟動下載，並透過 SSE 看到即時更新直到完成或失敗。
- [ ] `main_env.py` 與 API 使用同一個共用 runner（不重複 `Downloader` 建構邏輯）。
- [ ] 自動化測試涵蓋環境變數解析／驗證輔助函式與 SSE／事件格式（在實務上可不啟動瀏覽器者，以單元測試為主）。

## v1 範圍外

- 以資料庫儲存或同步設定（維持 `.env`）
- 多使用者、遠端存取、身分驗證
- WebSocket 傳輸
- 並行下載或完整工作佇列 UI
